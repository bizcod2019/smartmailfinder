# 内存优化配置文档

## 概述

本文档详细说明了为解决Vercel部署内存限制问题而实施的各项内存优化措施。这些优化旨在将内存使用量控制在512MB以内，确保应用能够在Vercel环境中稳定运行。

## 优化措施总览

### 1. 数据处理限制

#### 邮件数量限制
- **语义搜索引擎**: `max_emails` 从 1000 降至 500
- **轻量级搜索引擎**: `max_documents` 从 1000 降至 500
- **环境变量**: `MAX_EMAILS` 从 30000 降至 5000

#### 搜索结果限制
- **语义搜索**: `max_results` 从 500 降至 100
- **UI界面**: 搜索结果滑块范围从 10-100 (默认20) 改为 5-50 (默认10)

#### 批处理大小
- **语义搜索**: embedding生成批次大小从 50 降至 20
- **索引构建**: `INDEX_BATCH_SIZE` 从 500 降至 100

### 2. 文本长度限制

#### 邮件内容截断
- **组合文本**: 从 2000 字符降至 1000 字符
- **邮件正文**: 从 1000 字符降至 500 字符
- **HTML内容**: 从 1000 字符降至 500 字符
- **搜索预览**: 从 500 字符降至 300 字符，进一步降至 150 字符

### 3. 缓存优化

#### 缓存时间
- **Streamlit缓存TTL**: 从 300 秒 (5分钟) 降至 120 秒 (2分钟)

#### 索引时间片
- **INDEX_TIME_SLICE_SEC**: 从 20 秒降至 10 秒

### 4. 外部API集成

#### OpenAI搜索引擎
- 新增 `OpenAISemanticSearch` 类，使用外部API避免本地模型加载
- 零内存占用的embedding服务
- 自动降级到关键词搜索
- 支持环境变量 `OPENAI_API_KEY` 配置

## 配置文件更新

### .env.vercel.example
```bash
# 应用设置 - 优化后的内存限制
MAX_EMAILS=5000                    # 从 30000 降至 5000
INDEX_BATCH_SIZE=100              # 从 500 降至 100
INDEX_TIME_SLICE_SEC=10           # 从 20 降至 10

# OpenAI API (可选)
OPENAI_API_KEY=your_openai_api_key_here
```

### requirements.txt
```bash
# 新增OpenAI API支持
openai>=1.0.0
```

## 搜索引擎优先级

系统现在按以下优先级选择搜索引擎：

1. **OpenAI搜索引擎** (如果配置了API密钥)
   - 零内存占用
   - 高质量embedding
   - 适合生产环境

2. **轻量级搜索引擎** (默认)
   - 基于简单词向量
   - 低内存占用
   - 适合资源受限环境

3. **完整语义搜索引擎** (仅在非Vercel环境)
   - 使用SentenceTransformer和FAISS
   - 高内存占用
   - 适合本地开发

## 内存使用估算

### 优化前
- 邮件数据: ~30MB (1000封邮件 × 30KB)
- 模型加载: ~200MB (SentenceTransformer)
- FAISS索引: ~50MB
- 其他开销: ~100MB
- **总计**: ~380MB

### 优化后
- 邮件数据: ~15MB (500封邮件 × 30KB)
- 轻量级模型: ~5MB
- 简单索引: ~10MB
- 其他开销: ~50MB
- **总计**: ~80MB

## 性能影响

### 正面影响
- 内存使用量减少约80%
- 启动时间显著缩短
- 更稳定的Vercel部署

### 可能的权衡
- 搜索结果数量减少
- 文本截断可能影响搜索精度
- 缓存时间缩短可能增加计算频率

## 监控建议

1. **内存使用监控**
   - 使用 `psutil` 监控实际内存使用
   - 设置内存使用警报

2. **性能监控**
   - 监控搜索响应时间
   - 跟踪搜索结果质量

3. **错误监控**
   - 监控内存溢出错误
   - 跟踪API调用失败

## 进一步优化建议

1. **数据分页**: 实现搜索结果分页加载
2. **懒加载**: 按需加载邮件内容
3. **压缩存储**: 使用压缩算法存储邮件数据
4. **外部存储**: 将大型数据移至外部存储服务

```python
INDEX_BATCH_SIZE=500
INDEX_TIME_SLICE_SEC=20
```
